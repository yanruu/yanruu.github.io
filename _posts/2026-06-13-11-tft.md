다변량 LSTM은 변수를 모두 동등하게 입력으로 쌓았다. 하지만 실제로는 변수마다 "성격"이 다르다 — 시간이 지나도 안 변하는 정보(상점 위치), 미래에 이미 알고 있는 정보(공휴일 여부), 과거에만 관측되는 정보(실제 판매량)가 섞여 있다. **TFT(Temporal Fusion Transformer)** 는 이 차이를 구조적으로 반영하고, 동시에 어텐션으로 해석 가능성까지 제공하는 모델이다.

## 입력 변수를 세 가지로 구분한다

| 변수 유형 | 설명 | 예시 |
|-----------|------|------|
| Static Covariates | 시간이 지나도 변하지 않음 | 매장 ID, 지역 |
| Known Future Inputs | 미래 시점도 미리 알 수 있음 | 요일, 공휴일 여부, 프로모션 일정 |
| Observed Inputs | 과거에만 관측 가능, 미래는 모름 | 실제 판매량, 실제 기온 |

다변량 LSTM은 이 셋을 구분 없이 한 시퀀스로 섞었지만, TFT는 각 변수 유형을 별도로 받아 처리한다.

![TFT 구조 개요](/assets/images/tft_architecture_overview.png)
*세 종류의 입력이 Variable Selection Network로 들어가 "지금 어떤 변수가 중요한지" 가중치를 매기고, 이후 LSTM Encoder/Decoder와 Attention을 거쳐 분위수(Quantile) 형태로 출력된다.*

## Variable Selection Network (VSN)

VSN은 입력 변수마다 가중치를 학습해서, 시점별로 어떤 변수에 더 집중할지 동적으로 결정한다. 단순히 모든 변수를 합치는 것이 아니라, "이 시점에는 feature1이 중요하고, 다른 시점에는 feature3이 더 중요하다"는 식의 선택을 모델이 직접 학습한다.

## Gated Residual Network (GRN)

TFT 내부 곳곳에서 쓰이는 기본 빌딩 블록으로, 게이트로 정보를 통과시킬지 걸러낼지 조절하면서도 잔차 연결(residual connection)로 학습 안정성을 유지한다.

$$GRN(a) = LayerNorm(a + GLU(\eta_1))$$

여기서 $GLU$(Gated Linear Unit)가 "필요한 만큼만 통과시키는" 게이트 역할을 한다. 이 구조 덕분에 TFT는 깊게 쌓아도 학습이 잘 무너지지 않는다.

## Interpretable Multi-Head Attention

Transformer의 핵심인 어텐션을 시계열에 맞게 변형했다.

$$Attention(Q, K, V) = softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

일반 Transformer의 멀티헤드 어텐션과 달리 TFT는 헤드 간 가중치를 일부 공유해서, "이 예측이 과거 어느 시점을 가장 많이 참고했는지"를 사람이 해석할 수 있는 형태로 뽑아낼 수 있다. 이것이 LSTM 계열 대비 TFT의 가장 큰 장점 중 하나다 — 블랙박스가 아니라 "왜 이렇게 예측했는지"를 어느 정도 설명할 수 있다.

## 분위수(Quantile) 예측 — 불확실성까지 함께

TFT는 하나의 값만 예측하지 않고, q10/q50/q90처럼 여러 분위수를 동시에 예측한다. q50은 중간값(전형적인 예측), q10~q90 구간은 "90% 확률로 실제값이 이 범위 안에 있을 것"이라는 불확실성 추정이다.

```python
from pytorch_forecasting import TemporalFusionTransformer, TimeSeriesDataSet
from pytorch_forecasting.metrics import QuantileLoss

training = TimeSeriesDataSet(
    df_train,
    time_idx="time_idx",
    target="target",
    group_ids=["series_id"],
    static_categoricals=["store_id"],
    time_varying_known_reals=["day_of_week", "is_holiday"],
    time_varying_unknown_reals=["target", "feature1", "feature2"],
    max_encoder_length=30,
    max_prediction_length=7,
)

tft = TemporalFusionTransformer.from_dataset(
    training,
    learning_rate=0.01,
    hidden_size=16,
    attention_head_size=4,
    loss=QuantileLoss(),
)
```

`time_varying_known_reals`와 `time_varying_unknown_reals`를 나눠서 지정하는 부분이 앞서 설명한 "변수 유형 구분"이 코드로 드러나는 지점이다.

![분위수 예측 예시](/assets/images/quantile_prediction_example.png)
*검은 선이 실제값, 빨간 선이 q50(중간 예측), 음영이 q10~q90 구간이다. 시간이 지날수록 불확실성 구간이 넓어지는 모습도 ARIMA의 신뢰구간과 같은 맥락이다.*

## TFT vs LSTM

| 항목 | 다변량 LSTM | TFT |
|------|-------------|-----|
| 변수 유형 구분 | 없음 (모두 동등하게 입력) | Static/Known/Observed 구분 |
| 해석 가능성 | 낮음 | Attention으로 부분적 해석 가능 |
| 출력 형태 | 점 예측(point forecast) | 분위수 예측(불확실성 포함) |
| 구현 복잡도 | 낮음 | 높음 (변수 설계 필요) |

TFT는 강력하지만 변수 유형을 제대로 설계해야 성능이 나온다는 점에서 진입장벽이 있다. 다음 글에서는 이렇게 만든 모델을 "정말 믿을 수 있는지" 검증하는 방법, 백테스팅(Walk-forward Validation)을 다룬다 — 이 시리즈의 마지막 글이다.
